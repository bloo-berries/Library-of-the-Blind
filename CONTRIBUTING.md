# Contributing to Library of the Blind

Thank you for your interest in contributing to the Library of the Blind! This document provides guidelines for contributing to this comprehensive repository of blind and visually impaired communication methodologies.

## How to Contribute

### Reporting Issues
- Use GitHub Issues to report errors, suggest improvements, or request new entries
- Include as much detail as possible
- Reference existing documentation when relevant

### Adding New Entries
- Follow the existing documentation structure
- Use the taxonomy in `taxonomy.md` to determine where entries belong
- Include all relevant information:
  - Language/script covered
  - Key characteristics
  - Status (active, legacy, historical)
  - Origin/development history
  - Community/users
  - Related resources

### Updating Existing Entries
- Correct errors or outdated information
- Add missing details
- Improve clarity and organization
- Add references and resources

### Documentation Standards
- Use clear, accessible language
- Follow Markdown formatting conventions
- Include status indicators (active, legacy, historical)
- Provide context and background when relevant
- Cite sources when possible

## Categories

When adding content, ensure it fits into one of these categories:

1. **Braille Systems** - Language-specific Braille codes
2. **Alternative Tactile Scripts** - Non-Braille tactile writing systems
3. **Audio Channels** - Screen readers, audiobooks, TTS
4. **Tactile Modalities** - Refreshable displays, 3D models, tactile graphics
5. **Multimodal Systems** - Combined accessibility technologies
6. **Regional Accessibility** - Infrastructure by region

## Pull Request Process

1. Fork the repository
2. Create a feature branch
3. Make your changes following the documentation standards
4. Test that Markdown renders correctly
5. Submit a pull request with a clear description

## Questions?

If you have questions about contributing, please open an issue for discussion.

## Code of Conduct

- Be respectful and inclusive
- Acknowledge diverse perspectives and experiences
- Focus on creating comprehensive, accurate documentation
- Welcome contributions from all backgrounds
